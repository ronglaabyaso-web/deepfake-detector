# Prompt สรุปโปรเจกต์ "ตัวช่วยจับคลิปปลอม" (สำหรับพัฒนาต่อใน Claude Code)

> วิธีใช้: เปิดโฟลเดอร์ `deepfake-detector` ใน Claude Code แล้ววาง prompt นี้ เพื่อให้เข้าใจโปรเจกต์ทั้งหมดและพัฒนาต่อได้ทันที

---

## คำสั่งเริ่มต้น

นี่คือสรุปสถานะโปรเจกต์ทั้งหมด ช่วยอ่านให้เข้าใจภาพรวมก่อน แล้วทำงานต่อตามหัวข้อ "งานที่ต้องทำต่อ" โดย **รักษากฎในหัวข้อ "กฎสำคัญ" อย่างเคร่งครัด** ถ้าจุดไหนไม่ชัดเจนให้ถามก่อนแก้โค้ด

---

## 1. โปรเจกต์นี้คืออะไร

แอป (PWA) ชื่อ **"ตัวช่วยจับคลิปปลอม"** — เครื่องมือฟรีสำหรับคนไทย ใช้ตรวจ **คลิป / ลิงก์ / ข้อความ** ที่น่าสงสัยว่าเป็น Deepfake หรือมิจฉาชีพ

- **กลุ่มเป้าหมาย:** เด็ก ผู้สูงอายุ คนต่างจังหวัด → UI ต้องเรียบง่าย ตัวอักษรใหญ่ ปุ่มกดง่าย **ภาษาไทยล้วน ห้ามมีศัพท์เทคนิค** (API, model, inference ฯลฯ) ในส่วนที่ผู้ใช้เห็น
- **ใช้แข่ง** Samsung Solve for Tomorrow 2026 (หัวข้อ Cybersecurity) — ส่งเป็น **เดโมที่รันได้จริง**
- ต่อยอดจากโปรเจกต์ที่ชนะ NCSA Cybersecurity Product and Service Award 2025
- **ต้องฟรี 100%** ไม่มีค่าใช้จ่าย

---

## 2. เทคโนโลยีที่ใช้ (open source ทั้งหมด)

- **Backend:** FastAPI (Python 3.10–3.12) — เซิร์ฟเวอร์เดียวเสิร์ฟทั้ง API และหน้าเว็บ
- **Frontend:** PWA ไฟล์เดียว (vanilla JS + HTML + CSS) — **ไม่ใช้ React** (เคยมีปัญหา input หลุด focus)
- **AI (อธิบายผล/แชท/วิเคราะห์ข้อความ):** เลือกผู้ให้บริการได้ ค่าเริ่มต้นใช้ **Google Gemini ฟรี** (`gemini-2.5-flash`, SDK `google-genai`) — ถ้าไม่มี key ใช้โหมด "กฎพื้นฐาน" แทนได้ (ฟรี ไม่ต้องมี key)
- **ตรวจ Deepfake วิดีโอ:** โมเดล Hugging Face (image classifier) + OpenCV + yt-dlp — ฟรี รันในเครื่อง
- **ฐานข้อมูลผู้ใช้:** SQLite (มากับ Python)

## 3. สิ่งที่ทำเสร็จแล้ว (ทดสอบผ่านหมด)

**Backend**
- Endpoint ครบ: `GET /api/health`, `GET /api/scam-patterns`, `GET /api/golden-rules`, `POST /api/analyze-text`, `POST /api/analyze-video`, `POST /api/chat`, `POST /api/register`, `POST /api/login`, `POST /api/me`, `POST /api/logout`
- เสิร์ฟหน้าเว็บ (frontend) ที่ `/` ในตัว (รันเซิร์ฟเวอร์เดียวได้ทั้งแอป)
- ตัวเชื่อม AI หลายเจ้า (auto: gemini → claude → ollama → none) + โหมดกฎพื้นฐานเมื่อไม่มี key
- ฐาน **9 รูปแบบมิจฉาชีพ + กฎทอง 3 ข้อ**
- **ระบบสมาชิกในตัว** (สมัคร/เข้าสู่ระบบด้วยชื่อผู้ใช้+รหัสผ่าน, รหัสผ่าน hash ด้วย PBKDF2, session ใน memory)
- วิเคราะห์ข้อความ/SMS + แชทบอท (ล็อกขอบเขต + แนะนำสายด่วน 1441/191 เมื่อถูกหลอก) — ใช้งานได้จริง
- Pipeline วิเคราะห์วิดีโอ (ดาวน์โหลด→ตัดเฟรม→ตรวจหน้า→โมเดล→รวมผล) + degrade อย่างนุ่มนวลเมื่อ error
- ทุกผลวิดีโอแนบ `disclaimer_th` ว่าเป็น "สัญญาณเตือน ไม่ใช่คำตัดสิน"

**Frontend (6 หน้า + bottom nav)**
- **เข้าสู่ระบบ:** แท็บสมัคร/เข้าสู่ระบบ + ปุ่ม "เข้าใช้แบบผู้เยี่ยมชม" (ไม่มี Google/LINE แล้ว), จำสถานะล็อกอิน
- **หน้าหลัก:** ทางลัดไปตรวจคลิป/ข้อความ/แชท + กฎทอง
- **ตรวจสอบ:** สลับโหมด คลิป/ลิงก์ กับ ข้อความ/SMS, แสดง % ความเสี่ยง + จุดสังเกต + คำอธิบายไทย + disclaimer
- **เรียนรู้:** กฎทอง 3 ข้อ + รูปแบบมิจฉาชีพ (accordion) + **แบบทดสอบโต้ตอบ 7 ข้อ** (เฉลย+คำอธิบายทันที สรุปคะแนน ทำใหม่ได้)
- **ผู้ช่วย:** แชทบอท
- **ของฉัน:** ชื่อผู้ใช้, สายด่วน 1441/191, ตั้งค่าที่อยู่เซิร์ฟเวอร์, ออกจากระบบ
- Loading/error state ภาษาไทยที่เข้าใจง่าย, frontend ตรวจ origin เองแล้วเรียก API ที่เดียวกันอัตโนมัติ (ใช้ได้ทั้ง localhost/LAN/tunnel)
- เป็น PWA: มี manifest + service worker + ไอคอน (ติดตั้งลงเครื่องได้)

## 5. กฎสำคัญ (ต้องรักษาไว้ทุกครั้งที่แก้)

1. **บั๊ก input หลุด focus — ห้ามให้กลับมา** frontend ต้อง:
   - สร้าง DOM ของแต่ละหน้า **ครั้งเดียว** (ดู `built`/`ensureBuilt`) แล้วสลับหน้าด้วยการซ่อน/แสดง
   - **ห้าม** `innerHTML = ...` ทับ container ที่มี `<input>`/`<textarea>` ที่ผู้ใช้กำลังพิมพ์
   - อัปเดตเฉพาะ container ที่ไม่มี input (เช่น `#videoResult`, `#textResult`, `#chatMessages`, `#quizArea`)
   - ผูก event ด้วย `addEventListener` ครั้งเดียว ไม่ใช้ inline `onclick`
2. **ภาษาไทยล้วน ไม่มีศัพท์เทคนิคในส่วนที่ผู้ใช้เห็น**
3. **ผลตรวจ = "สัญญาณเตือน" ไม่ใช่คำตัดสิน 100%** — แสดง `disclaimer_th` เสมอ (งานวิจัย 2026 ยืนยันว่าโมเดล deepfake แม่นกับชุดที่ฝึกมา แต่พลาดกับของใหม่)
4. **ฟรี 100%** — AI ค่าเริ่มต้นใช้ Gemini ฟรี หรือโหมด none; **API key อยู่ฝั่ง backend เท่านั้น** ห้ามใส่ใน frontend
5. **ไม่ใช้ Google/LINE login** ใช้ระบบสมาชิกในตัว (ชื่อผู้ใช้+รหัสผ่าน) + มีโหมดผู้เยี่ยมชม
6. คงดีไซน์เดิม (ธีมมืด สีฟ้า #0ea5e9 การ์ดมุมโค้ง ปุ่มใหญ่กดง่าย)

## 6. วิธีรัน / ทดสอบ

```bash
cd backend
python -m venv venv
source venv/bin/activate           # Windows: venv\Scripts\activate
pip install -r requirements.txt     # หลัก (เบา) — พอสำหรับตรวจข้อความ+แชท+เรียนรู้
cp .env.example .env                # (ไม่บังคับ) ใส่ GEMINI_API_KEY ฟรีจาก aistudio.google.com/apikey
python run.py                       # เปิด http://localhost:8000
# อยากได้ตรวจคลิปวิดีโอ: pip install -r requirements-video.txt (ตัวหนัก)
```

หมายเหตุ: ไม่มี key ก็รันได้ (โหมดกฎพื้นฐาน) • โมเดลตรวจวิดีโอบน CPU ช้า มี GPU จะเร็วกว่ามาก

## 7. งานที่ต้องทำต่อ

### 7.1 (กำลังทำ — ทำต่อให้เสร็จ) เพิ่มความแม่นของการตรวจวิดีโอด้วย open source
ปรับ `backend/video_pipeline.py` ตามแนวทางนี้ (อิงงานวิจัย 2026: ไม่มีโมเดลเดี่ยวแม่นสุด ควรผสมหลายตัว + เน้นใบหน้า):
1. **Ensemble หลายโมเดล:** ให้ env `DEEPFAKE_MODEL_ID` รับหลายตัวคั่นด้วย comma → โหลดทุกตัว (ตัวที่โหลดไม่ได้ให้ข้าม) → ต่อเฟรม **เฉลี่ยคะแนน "ความเป็นของปลอม" จากทุกโมเดล**
   - ค่าเริ่มต้นแนะนำ: `prithivMLmods/deepfake-detector-model-v1,prithivMLmods/Deepfake-Detect-Siglip2`
   - หา index ของ label "fake" อย่างยืดหยุ่น (รองรับชื่อ label หลายแบบ / binary real-fake)
2. **วิเคราะห์เฉพาะเฟรมที่เจอใบหน้า** + ครอปหน้าแบบมีขอบ (~20%, env `FACE_MARGIN`) เพราะโมเดลฝึกจากใบหน้า; **ถ้าไม่เจอใบหน้าเลยทั้งคลิป → ตอบตรง ๆ ว่า "ไม่พบใบหน้า ประเมินไม่ได้"** (ไม่เดามั่ว)
3. คงการรวมผลแบบค่าเฉลี่ย + รายการ timestamp ที่น่าสงสัย + degrade เมื่อ error + ส่ง `models_used` กลับไปด้วย
4. เก็บ `MODEL_ID` (สตริงรวม) ไว้ให้ `/api/health` แสดงได้เหมือนเดิม

> (สถานะ: โค้ดส่วนนี้ออกแบบไว้แล้วแต่ยังไม่ได้บันทึกลงไฟล์ — ให้เขียนทับ `video_pipeline.py` ด้วยตรรกะข้างบน โดยคงฟังก์ชัน `analyze_video(url)` และคีย์ผลลัพธ์เดิม: `status, risk_percent, frames_analyzed, faces_found, suspicious_timestamps, error_th`)

**เพิ่มความแม่นได้อีก (ถ้ามีเวลา):** เปลี่ยนตัวตรวจใบหน้าจาก Haar cascade เป็น MediaPipe/InsightFace (แม่นกว่า), เพิ่มโมเดลแนว CLIP-based universal detector (generalize ดีกับของใหม่), ใช้ GPU เร่งความเร็ว

### 7.2 เตรียมเดโม
- เตรียม **คลิปตัวอย่าง 3–5 ลิงก์** ที่เทสต์ไว้แล้ว (กันโมเดล/เน็ตพลาดตอนแข่ง)
- ทำให้ติดตั้งเป็นแอปได้ (ดู INSTALL_APP.md): deploy frontend ฟรี + เปิด backend ด้วย cloudflared tunnel → ทำ QR ให้กรรมการสแกนติดตั้งบนมือถือ

### 7.3 ออปชันเสริม
- เสริมความปลอดภัยระบบสมาชิก (จำกัดจำนวนครั้งล็อกอิน, session ถาวร, hash แข็งขึ้น เช่น bcrypt/argon2), ป้องกัน endpoint ด้วย token
- แบบทดสอบ: สุ่มลำดับคำถาม / เพิ่มจำนวนข้อ / ทำเหรียญรางวัลเก็บความคืบหน้า
- เก็บประวัติการสแกนของผู้ใช้แต่ละคน

## 8. ข้อควรรู้ (สำคัญเชิงจริยธรรม)

โมเดลตรวจ deepfake สำเร็จรูป **แม่นกับชุดข้อมูลที่ฝึกมา แต่พลาดได้กับคลิปจริงนอกชุด/เทคนิคใหม่** กลุ่มเป้าหมายเชื่อง่าย ถ้าระบบมั่นใจผิดอาจทำให้เหยื่อวางใจผิดแล้วโอนเงิน → **ต้องกรอบผลเป็น "สัญญาณเตือนให้ระวัง" เสมอ และเน้นกฎทอง (พฤติกรรมมิจฉาชีพ) ควบคู่** ซึ่งเชื่อถือได้และช่วยเหยื่อได้จริงกว่าการเชื่อผลตรวจวิดีโออย่างเดียว
